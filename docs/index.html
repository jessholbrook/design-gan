Auto improving websites with agents.

design-gan is a dual-agent loop: a generator writes a single-page website from a brief, a critic scores it on the System Usability Scale, and the feedback rolls forward into the next draft — until the composite score plateaus.

The idea

Inspired by Andre Karpathy's Autoresearch, this experiment explores how that approach can be applied to basic website designs.

A generator agent writes a standalone HTML document from the brief. A headless Chromium takes a screenshot, dumps the DOM, and runs axe-core for an accessibility check. A critic agent then sees the rendered page — not the code — and fills out the 10-item System Usability Scale, writes feedback, and a short list of concrete next-step suggestions.

The suggestions feed the next generator call. The loop stops after a configurable window of iterations that produce no meaningful composite-score gain, or at a hard iteration cap.

How a single iteration works

brief ──▶ generator ──▶ HTML ──▶ headless Chromium
                                          │
                                          ├─▶ screenshot
                                          ├─▶ DOM
                                          └─▶ axe-core violations
                                                    │
                                                    ▼
                         critic ◀── System Usability Scale
                           │
                           ├─▶ SUS answers (10 × 1–5)
                           ├─▶ prose feedback
                           └─▶ prioritized suggestions ──▶ next generator call

The composite score is the standard SUS score (0–100) minus a weighted axe-core penalty. The accessibility penalty is capped — but it keeps the critic honest relative to visual appeal - e.g. a solid design can't hide a colour-contrast failure or missing alt text.

Design notes

  • The critic sees the rendered page, not the code. Code-only critique is cheap but correlates poorly with real usability.
  • Subjective + objective. SUS alone is gameable; axe-core anchors the score to measurable a11y signals.
  • Convergence, operationalised. “No further improvements” = patience iterations in a row without a composite gain of at least tolerance points.
  • Structured output. The critic uses Pydantic-validated JSON so a malformed response triggers a retry with a stricter nudge.
  • Prompt caching. System prompts are cached as ephemeral, so iteration N+1 pays near-zero for the static instructions.

Run it yourself

A live version is at design-gan.fly.dev — you can browse prior runs without anything, and start your own if you have the shared access token (ask the owner). Or run the whole loop locally: 

pip install -e .
playwright install chromium
cp .env.example .env  # add your ANTHROPIC_API_KEY

# Web UI — kick off runs, watch them stream in live
design-gan viewer      # http://127.0.0.1:8000

# Or one evolution loop from the terminal
design-gan run "A landing page for a weekend cycling tour in rural Vermont."

Source, setup details, and the Fly.io deploy config are in the GitHub repository.
